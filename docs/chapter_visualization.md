# 第X章 系统可视化模块设计

## 概述

为了直观地展示少样本学习模型的评估结果，帮助研究人员快速定位模型在不同家庭（Family）上的性能表现，本系统设计并实现了一套完整的可视化模块。该模块基于 Streamlit 框架构建，集成 Plotly 图表库，实现了从单Family细粒度指标到全局聚合统计的多层次可视化呈现。可视化模块作为系统评估流程的最终输出环节，承担着将复杂的数值计算结果转化为直观认知的关键职责。

---

## 6.1 可视化技术选型

### 6.1.1 前端框架选择

本系统采用 **Streamlit** 作为前端展示框架。Streamlit 是由 Snowflake 公司开发的一款开源 Python 框架，专门用于快速构建数据科学和机器学习领域的 Web 应用程序。与传统的 Web 开发技术（如 HTML、CSS、JavaScript）相比，Streamlit 具有以下显著优势：

首先，Streamlit 支持完全使用 Python 语言进行前端开发，开发人员无需掌握前端技术栈即可构建交互式数据应用。这对于本系统的主要用户群体——机器学习研究人员和算法工程师——而言，极大地降低了可视化模块的开发门槛。其次，Streamlit 内置了丰富的组件库，包括数据表格、图表、进度条、文件上传器等常用元素，开发人员可以通过简单的函数调用实现复杂的交互功能。此外，Streamlit 提供了自动状态管理机制，开发人员无需手动编写前端状态维护代码，即可在页面刷新或用户交互后保持页面状态的一致性。

在页面布局方面，Streamlit 支持多标签页（Tab）组织结构，本系统利用这一特性将四个核心功能模块分别置于不同的标签页中，用户可以通过点击标签页在数据集构建、特征提取、模型训练和结果可视化等功能模块之间自由切换。这种布局方式既保证了各功能模块的独立性，又在单一的 Web 页面中实现了完整的系统工作流程。

### 6.1.2 图表库选择

在图表渲染方面，本系统选用 **Plotly** 作为主要的图表绘制库。Plotly 是一个功能强大的交互式图表库，支持超过 40 种图表类型，包括折线图、柱状图、散点图、热力图、雷达图、仪表盘等多种可视化形式。Plotly 图表具有以下特性：

交互性是 Plotly 图表的核心优势。用户可以在图表上进行缩放、平移、悬停查看数据点详情等操作，这对于探索高维嵌入空间和大规模评估结果尤为实用。动态渲染能力使得图表可以根据用户输入实时更新，例如本系统中的 UMAP 可视化模块允许用户动态调整降维参数（邻居数、最小距离）并即时查看可视化结果。此外，Plotly 支持将图表导出为多种格式（PNG、SVG、HTML 等），便于研究人员在论文写作和成果展示时进行二次加工。

## 6.2 可视化模块架构设计

### 6.2.1 模块组成结构

可视化模块位于系统架构的顶层，直接面向最终用户。根据功能划分，本模块主要由以下几个可视化组件构成：

核心指标仪表盘（`plot_summary_gauges`）以仪表盘的形式展示模型评估的核心指标，包括总错误率、平均 EER、AUROC 等关键数值。仪表盘通过颜色区分指标的健康程度（绿色表示性能优良，橙色表示一般，红色表示较差），使用户能够一目了然地把握模型的总体表现。

Family级别评估图表（`plot_family_metrics`）以柱状图的形式展示各个 Family 的详细评估指标，包括错误率、EER、ID分类错误率、ID误拒率、OOD误接率等。该组件支持标准模式（仅展示错误率和EER）和 OOD 模式（展示完整的OOD相关指标）两种展示方式。

OOD 综合能力雷达图（`plot_ood_radar`）以雷达图的形式综合展示模型在 OOD 检测任务上的各项能力，包括 AUROC、1-FPR@95、OOD检测准确率、1-OOD误接率、1-ID误拒率等。该图表能够帮助研究人员快速识别模型在 OOD 检测方面的优势与不足。

UMAP 嵌入空间可视化（`plot_umap`）利用 UMAP 降维算法将高维说话人嵌入向量投影到二维空间，并以散点图的形式展示。通过颜色区分不同说话人，使用特殊标记标识 OOD 样本，该组件能够帮助研究人员直观地理解嵌入空间的分布特性以及模型对库外样本的识别能力。

相似度矩阵热力图（`plot_similarity_heatmap`）以热力图的形式展示测试样本与支撑集原型之间的相似度矩阵。通过颜色深浅表示相似度高低，该组件能够帮助研究人员分析模型在不同类别上的区分能力以及可能的混淆模式。

### 6.2.2 数据流转机制

可视化模块的数据来源于 Step 3（小样本学习与评估）模块的计算结果。当用户在 Step 3 中完成模型评估后，评估结果以字典形式存储在 Streamlit 的会话状态（Session State）中。每个 Family 的评估结果包含以下关键字段：

- `family_name`：Family 名称
- `err_rate`：错误率
- `eer`：等错误率
- `eer_threshold`：EER 对应阈值
- `auroc`（OOD 模式）：AUROC 指标
- `fpr_at_95`（OOD 模式）：FPR@95 指标
- `ood_err_rate`（OOD 模式）：OOD 误接率
- `id_class_err_rate`（OOD 模式）：ID 内部分类错误率
- `id_false_reject_rate`（OOD 模式）：ID 误拒率
- `similarity_matrix`：相似度矩阵（用于热力图和 UMAP 可视化）
- `true_labels`：真实标签（用于 UMAP 可视化）

当用户切换到 Step 4（结果可视化）标签页时，可视化模块从会话状态中读取评估结果，并调用相应的可视化函数生成图表。整个数据流转过程无需用户手动操作，实现了评估到可视化的无缝衔接。

## 6.3 核心可视化组件实现

### 6.3.1 核心指标仪表盘

核心指标仪表盘是用户进入结果可视化模块后首先看到的图表，它以高度聚合的方式展示模型的整体性能表现。仪表盘组件通过 `plot_summary_gauges` 函数实现，该函数接受评估指标字典和运行模式作为输入参数。

在标准模式下，仪表盘展示两个核心指标：总错误率（Total Error Rate）和平均 EER（Mean EER）。总错误率计算公式如下：

$$\text{Total Error Rate} = \frac{\sum_{i=1}^{N} \text{err\_num}_i}{\sum_{i=1}^{N} \text{test\_num}_i} \times 100\%$$

其中 $N$ 表示 Family 的总数，$\text{err\_num}_i$ 表示第 $i$ 个 Family 的错误样本数量，$\text{test\_num}_i$ 表示第 $i$ 个 Family 的测试样本总数。

平均 EER 的计算方式为各 Family EER 的算术平均值：

$$\text{Mean EER} = \frac{1}{N} \sum_{i=1}^{N} \text{EER}_i$$

在 OOD 模式下，仪表盘扩展为四个指标：ID 内部分类错误率、ID 误拒率、OOD 误接率和平均 AUROC。这些指标共同构成了评估开放集说话人验证系统性能的完整指标体系。

仪表盘的颜色策略采用阈值划分的方式。对于"越低越好"类型的指标（如错误率、EER），当指标值低于最大值的 30% 时显示绿色，介于 30% 到 60% 之间时显示橙色，高于 60% 时显示红色。对于"越高越好"类型的指标（如 AUROC），阈值划分方式相反。这种颜色编码机制借鉴了交通信号灯的语义，使用户能够快速判断各项指标的优劣。

### 6.3.2 Family 级别评估图表

Family 级别评估图表是进行细粒度分析的主要工具，它以柱状图的形式展示每个 Family 的具体评估指标。图表组件通过 `plot_family_metrics` 函数实现。

在标准模式下，图表包含两个子图：左侧子图展示各 Family 的错误率，右侧子图展示各 Family 的 EER。图表在每个子图的对应位置添加了一条水平虚线，用于标识该指标的全局均值。这种设计使得用户既能比较不同 Family 之间的性能差异，又能快速定位偏离均值较大的异常 Family。

在 OOD 模式下，图表扩展为 2×2 的子图布局，分别展示 ID 内部分类错误率、ID 误拒率、OOD 误接率和 EER。不同的指标子图采用不同的颜色方案：ID 内部分类错误率采用蓝色，ID 误拒率采用红色，OOD 误接率采用黄色，EER 采用绿色。这种颜色区分策略与仪表盘保持一致，便于用户在不同图表之间建立统一的认知关联。

图表的 X 轴为 Family 名称（按字母顺序排列），Y 轴为指标数值（百分比形式）。当 Family 数量较多时，X 轴标签会自动旋转以避免重叠，同时缩小字体尺寸以保证可读性。

### 6.3.3 OOD 综合能力雷达图

OOD 综合能力雷达图是专门为 OOD 检测任务设计的多维能力展示组件。雷达图通过 `plot_ood_radar` 函数实现，该函数接受汇总指标字典作为输入。

雷达图的五个维度定义如下：

1. **AUROC**：ROC 曲线下面积，衡量模型区分 in-domain 和 OOD 样本的整体能力
2. **1-FPR@95**：在 95% 真正率下的假正率的补，衡量模型在高召回率下的精确度
3. **OOD 检测准确率**：在最佳阈值下的二分类准确率
4. **1-OOD误接率/100**：OOD 误接率的补，衡量模型对库外样本的拒识能力
5. **1-ID误拒率/100**：ID 误拒率的补，衡量模型对库内样本的接受能力

所有维度的数值都被归一化到 [0, 1] 区间，其中 1 表示最佳性能，0 表示最差性能。雷达图通过填充区域直观地展示模型在各维度上的均衡性，填充区域越接近正五边形，表示模型在 OOD 检测的各项能力上越加均衡。

### 6.3.4 UMAP 嵌入空间可视化

UMAP（Uniform Manifold Approximation and Projection）是一种流行的非线性降维算法，能够在高维数据的局部和全局结构之间取得良好的平衡。本系统利用 UMAP 将说话人嵌入向量投影到二维空间，以散点图的形式展示嵌入空间的分布特性。

UMAP 可视化组件通过 `plot_umap` 函数实现，该函数接受嵌入矩阵、标签向量、可视化标题、UMAP 参数和 OOD 掩码作为输入。函数的实现流程如下：

首先，根据用户指定的邻居数（`n_neighbors`）和最小距离（`min_dist`）参数创建 UMAP 降维器。邻居数参数控制每个数据点在局部邻域中考虑的其他数据点数量，较小的值能够揭示数据的局部结构，较大的值则更注重全局结构。最小距离参数控制嵌入点之间的最小间距，较小的值会使嵌入点更加聚集，较大的值则会使嵌入点更加分散。

然后，使用 `fit_transform` 方法对嵌入矩阵进行降维，得到二维坐标。最后，根据标签向量为每个数据点分配颜色，并使用 Plotly 散点图进行渲染。

在颜色分配策略上，系统采用 Plotly 内置的多色调色板（包括 Plotly、Set2、Pastel 等），为每个唯一的标签分配一种独特的颜色。OD 样本采用特殊的灰色 X 标记进行标识，以区别于 in-domain 样本。这种设计使得用户能够直观地观察 OOD 样本在嵌入空间中的分布位置，判断模型是否能够将 OOD 样本与 in-domain 样本有效分离。

此外，图表支持交互式操作，用户可以通过鼠标悬停查看每个数据点的详细信息（坐标值、说话人 ID 等），通过框选操作放大感兴趣的区域，通过双击操作恢复默认视图。

### 6.3.5 相似度矩阵热力图

相似度矩阵热力图是分析模型分类决策过程的重要工具。在少样本学习场景中，模型通过计算测试样本与支撑集原型之间的相似度来进行分类决策。相似度矩阵直观地展示了这一决策过程，帮助研究人员理解模型在各类别上的区分能力。

热力图组件通过 `plot_similarity_heatmap` 函数实现，该函数接受相似度矩阵、行标签、列标签和标题作为输入。相似度矩阵的每一行对应一个测试样本，每一列对应一个支撑集原型，矩阵元素 $(i, j)$ 表示第 $i$ 个测试样本与第 $j$ 个原型的相似度。

在颜色映射方面，系统采用红蓝（RdBu）发散型色彩方案，其中红色表示高相似度，蓝色表示低相似度，0 值（无相似度差异）映射为白色。这种色彩方案便于用户识别模型在每个测试样本上的分类决策——如果某一行中某一列的颜色明显深于其他列，则表明模型将该测试样本预测为对应的类别。

当矩阵规模较小（行数 $\leq$ 20 且列数 $\leq$ 20）时，热力图会在每个单元格中显示具体的数值，便于精确分析。当矩阵规模较大时，数值注解被自动隐藏以避免视觉混乱，用户可以通过鼠标悬停查看单个单元格的数值。

图表的 X 轴标签为"支撑集（Prototype）"，Y 轴标签为"测试样本"。系统支持按真实标签排序测试样本，这一功能能够帮助用户在热力图中识别不同类别样本的聚集模式。

### 6.3.6 训练损失曲线

训练损失曲线用于展示模型在训练过程中的损失值变化趋势，帮助研究人员判断模型是否正常收敛以及是否需要调整超参数。损失曲线组件通过 `plot_loss_curve` 函数实现，该函数接受损失值列表、学习率和训练轮数作为输入。

损失曲线以训练轮次（Epoch）为 X 轴，以损失值为 Y 轴，采用折线图的形式展示。图表在曲线上添加了数据点标记，使曲线更加清晰。图表底部添加了一条水平虚线，标识训练过程中的最小损失值及其具体数值，便于研究人员快速定位最优训练轮次。

考虑到损失曲线通常在训练过程中实时生成，本系统在 Step 3 的训练循环中记录每个 Epoch 的损失值，并在训练完成后传递给可视化模块进行展示。

## 6.4 用户交互设计

### 6.4.1 实时进度展示

在模型评估过程中，系统通过 Streamlit 的进度条和实时日志展示训练进度。进度条以百分比的形式展示当前处理的 Family 占总 Family 数量的比例，日志区域滚动展示每个 Family 的处理状态和关键输出。这种设计使用户能够监控长时间运行的评估任务，及时发现和处理异常情况。

评估完成后，系统会在页面顶部显示"评估完成"的提示信息，并展示已处理的 Family 数量和当前平均 EER 等汇总指标。这些信息帮助用户快速判断评估是否成功完成以及模型的整体表现。

### 6.4.2 参数动态调整

部分可视化组件支持用户动态调整参数。以 UMAP 可视化为例，系统在图表旁边提供了两个滑块控件，分别用于调整 UMAP 的邻居数（取值范围 2-20，默认值 8）和最小距离（取值范围 0.05-0.80，默认值 0.30）。用户调整参数后，点击"绘制 UMAP"按钮即可重新生成降维结果和可视化图表。

这种交互设计使用户能够在不重新运行评估流程的情况下探索不同的可视化效果，降低了分析成本，提高了研究效率。

### 6.4.3 历史结果加载

为了支持对历史评估结果的回顾和分析，系统提供了历史结果文件加载功能。用户可以通过文件上传器导入之前保存的评估结果文件（支持 `.txt` 格式），系统会自动解析文件内容并提取关键指标，以数据表格和柱状图的形式展示。

解析功能支持两种结果文件格式：`few_shot_learning_result*.txt` 和 `baseline_result*.txt`。系统通过正则表达式从文件内容中提取"错误率为: XX% 和"EER为: XX%"等模式匹配的结果数值，转换为结构化的 DataFrame 格式进行展示。

### 6.4.4 结果导出

评估结果支持导出为 CSV 格式，方便用户进行二次分析或与其他系统对接。导出按钮位于 Family 详细结果表格的下方，点击后浏览器会下载一个以 `sv_results_{mode}_{timestamp}.csv` 命名的文件，其中包含所有 Family 的详细评估指标。

CSV 文件的列根据运行模式动态调整：标准模式包含 Family、错误率(%)、EER(%)、EER阈值等字段；OOD 模式额外包含 AUROC、FPR@95、OOD误接率(%)、ID误拒率(%)等字段。

## 6.5 本章小结

本章详细介绍了开放集说话人识别系统的可视化模块设计与实现。该模块基于 Streamlit 和 Plotly 技术栈，构建了一套从宏观到微观、从静态到动态的完整可视化体系。

在技术选型方面，Streamlit 框架以其简洁的 Python API 和强大的状态管理能力，为本系统的可视化开发提供了高效的基础设施。Plotly 图表库的交互性和动态渲染能力，使得复杂的评估结果能够以直观且可探索的方式呈现给用户。

在功能设计方面，本模块实现了六种核心可视化组件：核心指标仪表盘提供全局性能概览，Family 级别评估图表支持细粒度分析，OOD 综合能力雷达图展示多维能力均衡性，UMAP 嵌入空间可视化揭示高维特征分布，相似度矩阵热力图分析分类决策过程，训练损失曲线监控模型收敛状态。这些组件相互配合，共同构成了完整的评估结果可视化解决方案。

在用户交互方面，本模块注重实操性和灵活性。实时进度展示使用户能够监控评估任务，参数动态调整支持探索性分析，历史结果加载便于回顾和比较，结果导出便于二次处理。这些交互设计使得可视化模块不仅是一个结果展示工具，更是一个功能完备的数据分析平台。

通过本章设计实现的可视化模块，研究人员可以直观地了解模型在不同 Family 上的性能表现，分析 OOD 检测能力的优势和不足，探索嵌入空间的分布特性，从而为模型优化和算法改进提供有力的数据支撑。
